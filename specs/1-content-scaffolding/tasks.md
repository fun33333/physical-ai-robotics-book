## Dependencies & Execution Order

### Phase Dependencies

- **Setup (Phase 1)**: No dependencies - can start immediately
- **Foundational (Phase 2)**: Depends on Setup completion - BLOCKS all user stories
- **User Stories (Phase 3-8)**: All depend on Foundational phase completion
  - US1 (Introduction): Can proceed immediately after Phase 2
  - US2 (Modules): Can proceed in parallel with US1
  - US3 (Hardware): Can proceed in parallel with US1/US2
  - US4 (Weekly): Can proceed in parallel with US1/US2/US3
  - US5 (Assessments): Can proceed in parallel with all above
  - US6 (Build Validation): Depends on US1-US5 completion
- **Polish (Phase 9)**: Depends on all user stories being complete

### User Story Dependencies

- **User Story 1 (P1)**: Can start after Foundational (Phase 2) - No dependencies on other stories
- **User Story 2 (P2)**: Can start after Foundational (Phase 2) - No dependencies on US1
- **User Story 3 (P3)**: Can start after Foundational (Phase 2) - No dependencies on US1/US2
- **User Story 4 (P4)**: Can start after Foundational (Phase 2) - No dependencies on US1/US2/US3
- **User Story 5 (P5)**: Can start after Foundational (Phase 2) - No dependencies on US1-US4
- **User Story 6 (P6)**: DEPENDS on US1-US5 completion (build validation requires all content)

### Within Each User Story

- All markdown file creation tasks within a story are parallelizable
- Index files should ideally be created before chapter files (but not strictly required due to autogeneration)

### Parallel Opportunities

- All Setup tasks can run sequentially (T001 ‚Üí T002)
- Foundational tasks T003, T004 modify the same file - run sequentially
- Once Foundational phase completes, US1-US5 can start in parallel
- Within each module in US2, all chapter files can be created in parallel
- Within US3 (Hardware), all pages can be created in parallel
- Within US4 (Weekly), all week pages can be created in parallel
- Within US5 (Assessments), all pages can be created in parallel (after directory creation)

3. Add User Story 2 (All Modules) ‚Üí Test ‚Üí Deploy

1. Complete Setup + Foundational together
