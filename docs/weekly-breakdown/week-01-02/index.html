Week 1-2: ROS 2 Fundamentals

Overview​

Weeks 1-2 establish the foundation of ROS 2, covering core concepts, installation, and basic communication patterns. Students will build their first ROS 2 nodes and understand the system architecture.

Learning Objectives​

By the end of this 2-week period, students will be able to:

  • Install and configure ROS 2 Humble Hawksbill
  • Create and run basic ROS 2 nodes in Python
  • Implement the three core communication patterns: topics, services, and actions
  • Use ROS 2 tools for debugging and introspection
  • Structure ROS 2 packages appropriately

Time Estimate​

  • Total Time: 40 hours (20 hours per week)
  • Theory: 8 hours
  • Hands-on Lab: 24 hours
  • Project Work: 8 hours

Week 1: ROS 2 Architecture & Nodes​

Day 1: Installation & Environment Setup (4 hours)​

Morning (2 hours)​

  • Install ROS 2 Humble on Ubuntu 22.04
  • Configure development environment
  • Verify installation with basic tests

Afternoon (2 hours)​

  • ROS 2 workspace setup
  • Create your first ROS 2 package
  • Basic publisher/subscriber example

Daily Activities​

  1. Follow the ROS 2 installation guide
  2. Create a robotics_ws workspace
  3. Build and source the workspace
  4. Create your first package: my_robot_pkg

Checkpoint Criteria​

  • ROS 2 environment variables are properly configured
  • Successfully run ros2 run demo_nodes_cpp talker and ros2 run demo_nodes_py listener

Resources​

  • ROS 2 Humble Installation Guide
  • ROS 2 Tutorials

Day 2: ROS 2 Nodes Deep Dive (4 hours)​

Morning (2 hours)​

  • Understanding ROS 2 nodes architecture
  • Lifecycle of a ROS 2 node
  • Node parameters and configuration

Afternoon (2 hours)​

  • Create custom ROS 2 node in Python
  • Implement node parameters
  • Use rclpy to create nodes with proper structure

Daily Activities​

  1. Write a custom node that publishes sensor data
  2. Implement parameters for node configuration
  3. Use the lifecycle node pattern for robust operation

Checkpoint Criteria​

  • Your custom node successfully publishes messages to a topic
  • Node parameters can be set at launch time
  • Node properly handles shutdown signals

Resources​

  • Nodes Documentation
  • Parameters Tutorial

Day 3: Topics Communication (4 hours)​

Morning (2 hours)​

  • Understanding topics and message passing
  • Message types and custom message creation
  • Publisher/subscriber patterns

Afternoon (2 hours)​

  • Implement publisher and subscriber nodes
  • Create custom message types
  • Verify message passing between nodes

Daily Activities​

  1. Create a custom message for robot sensor data
  2. Implement publisher that sends sensor readings
  3. Implement subscriber that processes sensor data
  4. Use ros2 topic tools for debugging

Checkpoint Criteria​

  • Custom message definition compiles successfully
  • Publisher and subscriber nodes communicate properly
  • Messages can be monitored using ros2 topic echo

Resources​

  • Topics Documentation
  • Custom Messages Tutorial

Day 4: Services & Client Communication (4 hours)​

Morning (2 hours)​

  • Understanding services vs topics
  • Service definitions and synchronous communication
  • When to use services vs topics

Afternoon (2 hours)​

  • Implement service server and client
  • Handle service requests and responses
  • Error handling in services

Daily Activities​

  1. Create a custom service for robot commands
  2. Implement service server that handles commands
  3. Implement client that sends requests
  4. Test service reliability and error handling

Checkpoint Criteria​

  • Service calls complete successfully
  • Error handling works appropriately
  • Service can be monitored using ROS 2 tools

Resources​

  • Services Documentation
  • Services Tutorial

Day 5: Week 1 Project & Review (4 hours)​

Morning (2 hours)​

  • Integrate week's concepts into a complete node
  • Combine publishers, subscribers, and services
  • Test communication patterns together

Afternoon (2 hours)​

  • Debug and troubleshoot any issues
  • Prepare documentation for your node
  • Peer review session with classmates

Daily Activities​

  1. Build a complete "robot_status" node
  2. Use topics for sensor data, services for commands
  3. Document the node architecture
  4. Prepare for Week 2 concepts

Checkpoint Criteria​

  • All communication patterns from Week 1 integrated
  • Node successfully passes integrated testing
  • Documentation is complete and clear

Week 2: Advanced ROS 2 Concepts​

Day 6: Actions (4 hours)​

Morning (2 hours)​

  • Understanding actions vs services
  • Asynchronous communication with feedback
  • When to use actions vs services

Afternoon (2 hours)​

  • Implement action server and client
  • Handle long-running operations
  • Provide feedback during execution

Daily Activities​

  1. Create an action for robot navigation goals
  2. Implement action server with feedback
  3. Implement action client that monitors progress
  4. Test cancellation and preemption

Checkpoint Criteria​

  • Action server handles goals with feedback
  • Action client can send goals and monitor progress
  • Cancellation functionality works correctly

Resources​

  • Actions Documentation
  • Actions Tutorial

Day 7: Launch Files & System Management (4 hours)​

Morning (2 hours)​

  • Understanding launch files and system composition
  • Parameters and configuration management
  • Launch file syntax and best practices

Afternoon (2 hours)​

  • Create launch files for multi-node systems
  • Parameter files and command line parameter passing
  • Debug systems with launch files

Daily Activities​

  1. Create a launch file for your robot system
  2. Use parameters to configure nodes
  3. Launch multiple nodes with dependencies
  4. Test system startup and shutdown

Checkpoint Criteria​

  • Launch file successfully starts all required nodes
  • Parameters are properly managed and passed
  • System handles errors gracefully

Resources​

  • Launch Files Documentation
  • Launch Tutorial

Day 8: Tools & Debugging (4 hours)​

Morning (2 hours)​

  • ROS 2 command-line tools (ros2, rqt)
  • Visualization tools (RViz2)
  • Performance monitoring

Afternoon (2 hours)​

  • Debugging techniques for ROS 2 systems
  • Network configuration and multi-robot systems
  • Logging and introspection

Daily Activities​

  1. Use rqt to visualize your system
  2. Monitor topics and services with tools
  3. Debug communication issues
  4. Set up logging for your nodes

Checkpoint Criteria​

  • System can be properly monitored with tools
  • Debugging techniques are effective
  • Logging provides useful information

Resources​

  • Tools Documentation
  • RViz2 Tutorial

Day 9: Package Organization & Best Practices (4 hours)​

Morning (2 hours)​

  • ROS 2 package structure and conventions
  • CMakeLists.txt and package.xml
  • Code style and documentation

Afternoon (2 hours)​

  • Testing strategies for ROS 2 packages
  • CI/CD for ROS 2 development
  • Publishing and sharing packages

Daily Activities​

  1. Organize your code into proper package structure
  2. Write unit tests for your nodes
  3. Document your package with README and Doxygen
  4. Set up basic CI testing

Checkpoint Criteria​

  • Package follows ROS 2 conventions
  • Tests pass reliably
  • Documentation is complete and clear

Resources​

  • Package Structure Documentation
  • Testing Tutorial

Day 10: Week 2 Project & Assessment (4 hours)​

Morning (2 hours)​

  • Complete integrated project incorporating all Week 2 concepts
  • Implement complex communication patterns
  • Test with launch files and system tools

Afternoon (2 hours)​

  • Comprehensive system testing
  • Peer review and feedback
  • Prepare for Week 3 (Simulation)

Daily Activities​

  1. Build a complete robot simulation interface
  2. Combine all communication patterns (topics, services, actions)
  3. Use launch files and parameters effectively
  4. Document and present your system

Checkpoint Criteria​

  • Complete ROS 2 system with all communication patterns
  • System properly configured with launch files
  • All components tested and documented

Assessment​

Knowledge Check​

Complete the Week 1-2 Quiz to verify your understanding of ROS 2 fundamentals.

Practical Assessment​

Submit your complete robot node system including:

  • Node implementation with all communication patterns
  • Launch files for system startup
  • Test cases and documentation
  • Peer review feedback incorporation

Self-Assessment Questions​

After completing this 2-week period, you should be able to answer:

  1. Explain the ROS 2 architecture and how nodes communicate
  2. Create custom messages, services, and actions
  3. Use launch files to manage multi-node systems
  4. Debug common ROS 2 issues using tools
  5. Structure code following ROS 2 best practices

Going Further​

Advanced Topics​

  • Lifecycle nodes for robust state management
  • Composition for better performance
  • Real-time considerations
  • Security features
