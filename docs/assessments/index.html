Assessments

Assessment is integral to learning, providing feedback on your progress and ensuring you're building solid foundations before advancing. This section outlines our assessment philosophy and structure.

Assessment Philosophy​

We emphasize practical, hands-on assessments over theoretical exams. The primary assessments are projects that demonstrate real skills, supplemented by knowledge checks that verify conceptual understanding. This approach mirrors how skills are evaluated in industry and research.

Assessments are designed to be learning opportunities, not just evaluation gates. Each assessment includes detailed rubrics, example solutions, and guidance for self-improvement.

Grade Distribution​

ComponentWeightDescription
Knowledge Checks15%Weekly quizzes on concepts
Mini-Projects35%Bi-weekly coding projects
Capstone Project50%Final integration project

Assessment Types​

Knowledge Checks​

Short quizzes testing conceptual understanding:

  • Format: Multiple choice, short answer
  • Frequency: End of each week pair (Weeks 1-2, 3-4, etc.)
  • Duration: 20-30 minutes
  • Attempts: Unlimited (highest score counts)
  • Purpose: Verify core concept understanding

View Knowledge Checks

Mini-Projects​

Hands-on coding projects demonstrating practical skills:

  • Format: Code submission with documentation
  • Frequency: End of each two-week module
  • Duration: 4-8 hours of work
  • Submission: GitHub repository
  • Purpose: Apply concepts to real problems

View Projects

Capstone Project​

Comprehensive integration project:

  • Format: Complete Physical AI system
  • Duration: Week 8 (full week)
  • Submission: Code, report, video, presentation
  • Purpose: Demonstrate end-to-end mastery

View Projects

Grading Criteria​

Technical Competence (40%)​

  • Correct implementation of required features
  • Code quality and best practices
  • System reliability and robustness
  • Performance optimization

Problem Solving (25%)​

  • Approach to debugging and troubleshooting
  • Handling edge cases
  • Recovery from failures
  • Creative solutions

Documentation (20%)​

  • Code comments and README
  • Architecture documentation
  • Setup instructions
  • Clear explanations

Presentation (15%)​

  • Demo quality
  • Clear communication
  • Honest assessment of limitations
  • Response to questions

Academic Integrity​

Collaboration Policy​

  • Encouraged: Discussing concepts, helping debug, sharing resources
  • Not Allowed: Copying code, sharing solutions, using unauthorized AI

Attribution Requirements​

  • Cite all external code sources
  • Document AI assistant usage
  • Acknowledge peer help

Consequences​

  • First offense: Warning and resubmission
  • Second offense: Zero on assignment
  • Third offense: Course failure

Getting Help​

Before Assessment​

  • Attend office hours
  • Post questions in discussion forum
  • Review example solutions from previous modules

During Assessment​

  • Knowledge checks: Use course materials
  • Projects: Ask clarifying questions only
  • Capstone: Full instructor support available

After Assessment​

  • Review feedback carefully
  • Ask for clarification on unclear points
  • Use feedback to improve future work

Accommodations​

If you require accommodations:

  1. Contact course administration
  2. Provide documentation as needed
  3. Extended time available on request
  4. Alternative formats can be arranged
